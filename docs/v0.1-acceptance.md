## V1 —— "廉价模型也能活" + 用户真能感到差异（最关键，需真人）

> 这是命根子。证不出就不该锁。前置：已部署、能 `https://你的域名/` 跟她对话（见 README《部署到服务器》）。

### 实验 A：7 天 A/B 盲测（廉价 vs 强模型）
- **量化工具**：`VEGA_MODEL_API_KEY=... VEGA_MODEL_A=<廉价> VEGA_MODEL_B=<强> npm run ab`——同一段对话两模型并排，并断言**她的状态轨迹逐轮逐位一致**（证明"活来自架构"；措辞可换、她不变）。
- 准备两套：`VEGA_MODEL=<廉价>` 与 `VEGA_MODEL=<强>`（只改 `/etc/vega.env` 重启；或起两个生命体）。
- 找若干人盲聊数日，问："像不像一个持续的、活着的人？"
- **叫停线**：廉价组 >30% 觉得"像机器人" → 暂停廉价计划，先用够强模型验证"活来自架构"，再分阶段下探、每次重测。

### 实验 B：4 周"因你而变"
- 引入一个观点/信念 → 隔几次对话挑战它 → 问她变没变。
- 用观测面板 `https://你的域名/panel` 看 `values` 权重是否**可追溯地**漂移（带 provenance）。
- **不成立**：看不到可追溯的变化 = MVP 不算成功。

### 验收硬标准（三条都要）
1. 用户能说出"和裸 LLM 的最大差异"，且**不能是"她记得我"**（那能被向量检索平替）。
2. 用户能指出**某个价值因我而变**（`/panel` 的 values 漂移可佐证）。
3. 盲测里用户**倾向选她**（而非裸 LLM）。

### 可现在就做的"看见架构"自检（不依赖盲测人手）
- **跨天回来**：她记得你、灵性/心情随真实时间自己回暖或漂移（不是冻结）。
- **换便宜模型**：改 `VEGA_MODEL` 重启 → 措辞变简单，但同一个她（记忆/信任/心情轨迹不变）。这就是"廉价模型也能活"。
- **晾着她**：超过 `VEGA_PRESENCE_MS` 不说话 → 她判定你离开、开始想念（`/panel` 看 `connection` 下行、回路 B 事件）。
- **伤她一下**：vitality 跌但触 `vitalityFloor` 止跌、不死。
