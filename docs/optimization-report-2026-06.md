## 二、待修（按优先级）

### P0 — 影响核心体验，建议尽快

1. **微信回信真实送达（需线上联调）** — 代码已加送达检查+重发+日志，但 `ilink.ts` 这套是按文档写、**未在真机验证过**。下一步：线上发一条微信消息，看 `journalctl -u vega | grep sendmessage` 打印的真实返回，据此把 `sendMessage` 的 payload（message_type/state、context_token、to_user_id）改对。这是"微信收不到"的最终闭环。
   - 文件：`src/platform/ilink.ts:63`、`src/server/daemon.ts:355`。

2. **新用户引导缺失（首页结构/引导）** — 注册后直接落到「此刻」信息流，没有"先认识第一个她"的引导；`Onboarding.svelte` 只有 3 张预期说明页，进站后无任何指引。新用户不知道该干什么。
   - 建议：注册后引导到「探索 · 认识她们」并高亮"挑一个开始聊"；或首页空态给一步引导卡。

### P1 — 完整度/一致性，明显可感

3. **详情页头部不统一** — `PostDetail`、`LifeProfile`、`Chat` 三个二级页头部各写各的（有的 PageHeader、有的裸 `.ht`/`.htitle`、blur 有无不一）。
   - 建议：抽一个 `DetailHeader`（返回键 + 居中标题 + 可选 blur），三页共用。文件：`PostDetail.svelte:36`、`LifeProfile.svelte:24`、`Chat.svelte:90`。

4. **列表行样式三套并存** — 「探索·认识她们」用卡片 `.dossier`（圆角+边框），「对话」「通知」已改 X 风通栏行。三处行内边距也不一致（13px vs 16px）。
   - 建议：统一为一套"通栏行"（avatar + 主/次两行 + 右侧时间/状态），抽成 `ListRow` 组件，全站复用。文件：`Explore.svelte`、`Chats.svelte`、`Notifications.svelte`。

5. **生命体个人页（LifeProfile）设计偏弱** — 信息密度低、缺少"她最近在想什么/和谁更近/此刻在做什么"的叙事感；同类朋友与公开心声排版较平。标题重复已修。
   - 建议：hero 下加"此刻一句话状态"，同类朋友改横向头像条，公开心声做成时间线；补"她与你的关系"入口（若已认识）。

6. **加载态缺失** — `Chat`、`Me` 进入时无骨架屏，其它页有；体验不一致（"页面加速/快速响应"感知差)。
   - 建议：两页补 `Skeleton`。文件：`Chat.svelte:71`、`Me.svelte:23`。

7. **错误被静默吞掉，难定位线上问题** — 心跳/社交/世界回路与检查点/备份多处 `catch { /* ignore */ }`，线上出问题无任何信号（你这几天调微信吃过这个亏）。
   - 建议：运营关键路径的 catch 至少 `console.warn` 带上下文。文件：`daemon.ts` 多处（checkpoint 401-405、社交/评论/世界回路、备份）。
