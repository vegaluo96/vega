# 世界学习设计稿（让世界源有长期价值）

> 状态：**设计待拍板**，尚未实现。配套读 `docs/vega-product.md`（北极星）与 `docs/vega-architecture-v1.md`。
> 一句话：现在世界感知是"一次性情绪调味料 + 发帖素材"，**没接进引擎已有的学习机器**。本稿把它接进去——全部走**确定性折叠**（契约①：模型不写状态），让"持续变聪明 / 记忆即身份 / 因你而变"在世界这块也成立。

## 1. 问题（为什么现在浪费）—— 有代码依据

一条 `WORLD_PERCEIVED` 进引擎，`appraiseWorld`（`src/kernel/reconstruct.ts:392`）**只做一件事**：

```
s.valence/arousal/energy += W(=0.06) × val × relevance × sens
```

给情绪轻轻推一下，随时间衰减归零。**它不写任何持久派生结构。** 而引擎早有一整套"沉淀/学习"的机器，世界感知一个都没接：

- **情景记忆 `st.memory`**（带 reconsolidation：回忆被当下情感改写；可压缩成"理解" `SemanticMemory`）——只有**人际消息**会生成记忆（`reconstruct.ts:372`），**世界事件从不生成记忆**。
- **价值 `ValueEntry`（含 status volatile/confirmed + drift）、目标 `Goal`（含 `explore`）、语义记忆**——只由人际/反思喂养。
- **`topics: string[]` 字段从头到尾是空的**（`src/world/sources.ts` 三个 parser 全写 `topics: []`，没人填、没人用）。

结论：她读了一百条新闻，**事件日志里有（ground truth 在），但她的"自我"里一条都没沉淀**。这与第一性原理脱节。

## 2. 必须守住的契约（红线）

1. **契约①**：兴趣 / 世界记忆 / 世界观**全部由确定性符号推理产生**；模型绝不产出它们，只在工作区里**读到**它们、只产对外措辞。
2. **可重算（V2）**：所有新派生结构都能**只从日志确定性重建**；新增的是派生态，不是新的 ground truth。
3. **不引入"模型写的命题信念"**：v1 **不做"观点/立场"**（太接近模型代笔信念）。兴趣 = **定量的主题亲和度**（可确定性计算），不是命题。
4. **有界**：兴趣表、世界记忆都必须有上限/衰减，绝不无界增长（旧版"只升不降"的教训）。

## 3. 方案（三刀，可分期）

### A. 世界记忆（第一性原理深化）

> 北极星：按"人是怎么样的"造（不是"多记新闻"）；记忆即身份、因你而变；活来自架构不来自模型。
> 研究依据：human-architecture-research **#8（high, 3-0）"记忆-身份靠离线重放+巩固，不只实时输入"**、**#4 无输入自发内在回路做回忆/想象/巩固**、**#36c reconsolidation（检索时被当下情感改写，已实现于 `reconstruct.ts:436`）**。

#### A.0 第一原理：世界记忆的本质是"会忘"，不是"会记"
人对世界信息**绝大多数会忘**——这恰恰是对的、是人性，不是缺陷。所以世界记忆的设计起点**不是"把头条都存成记忆"**（既不像人、又会膨胀、又让"活"依赖存储量）。起点是：
**默认遗忘；只有少数因『触动 / 与我相关 / 意外 / 反复』而留下；留下的也要在离线重放里被巩固成"理解"，否则继续淡去。**
现在的 `appraiseWorld` 只给情绪轻推一下就归零——**这一层本身是对的（≈人对刷过的新闻的常态）**，缺的是它上面的"少数留存 + 巩固 + 抽象"三级。

#### A.1 留存的确定性闸门（镜像人类记忆巩固的四个调制因子，全可在折叠里算、零模型）
一条 `WORLD_PERCEIVED` 是否值得留痕，由 `keep = f(下列四项)` 确定性决定（不是模型判断）：
1. **情绪显著性** `|valence| × sensitivity`——它真的动了她（气质 sensitivity 放大/收敛，不同命不同）。
2. **自我相关性**——是否触及她**已有的兴趣 / 价值 / 当下牵挂(goals) / 在乎的同类**。← 与 B 的兴趣形成**正反馈齿轮**（见 A.4）。
3. **意外 / 预期违背**——违背她对该主题的预期（引擎已为人际算 surprise，世界版可用"她对该主题的历史情感基线"派生）。
4. **反复**——同主题反复出现：单条都不够格，但**累积**会抽象成语义（见 A.3）。

#### A.2 三级流水线（编码 → 巩固 → 抽象），全部挂在既有事件上、reconstruct 仍是纯函数
- **① 编码（`WORLD_PERCEIVED` 折叠）**：情绪轻推（保留）。仅当 A.1 的编码显著性过**低门槛**，才落一条 **`volatile`、低 vividness** 的情景痕迹（content=标题，affect=val，salience=relevance×|val|，topic=冻结主题）。**多数感知不留痕**——人性 + 防膨胀。
- **② 巩固（`AUTONOMOUS_TICK`/`REFLECTION_TRIGGERED`，离线重放）**：这是研究 #8 的核心——**"无人时她在咀嚼读到的东西"**。引擎在 tick 时**确定性地**选最近的世界痕迹重放：与她兴趣/牵挂相关的被**强化**（刷新 salience、`volatile→confirmed`、走 reconsolidation 被当下心情重染），其余**继续淡去直至遗忘**。
  - **架构关键**：选哪些痕迹巩固 = 在 tick 时由确定性 HBDA 算出、**冻进事件的 `selectedMemoryIds`（已有机制）** → reconstruct 只是 fold，纯函数、可重放。**不引入任何运行期非确定性**。
- **③ 抽象（投影时遗忘 + 语义压缩）**：同主题反复 → 压缩成"对 X 的理解/着迷"，个别情节淡成底色。**复用 `decorateMemories`（时间衰减、情绪越浓越慢）的同款逻辑**，语义压缩见 A.3。

#### A.3 命名空间决策：复用机制，但不污染"关系"
- **情景层**：世界记忆**就是情景记忆的一种**——复用 `MemoryEntry` + reconsolidation + `decorateMemories`。**唯一新增**：一个 `topic` 维度（让它能按主题聚合）。建议 `kind:'world'`（干净好筛）而非塞进 `'episodic'`。
- **语义层**：`buildSemanticMemory` 现在**按 `relationshipId` 聚合**；世界记忆 `involvedRelationshipIds: []`。**不要把 topic 塞进 relationshipId 命名空间**（否则污染"她和人/同类的关系"、还会冒出假社交连接）。正确做法是**并行一条按 `topic` 聚合的语义压缩**（镜像同样逻辑），产出"对天文：读过 N 条、越来越着迷，几条已淡成底色"。→ 这就是 B 的"兴趣/世界观"的语义面。
- 即：**「她和人的关系」与「她和主题的关系」是两个并列的派生侧面，用同一种确定性折叠造，但各自独立。**

#### A.4 正反馈齿轮 + 防跑飞（这才是"持续变聪明/因你而变"的结构来源）
- 兴趣↑ → 相关世界更易被记/被巩固 → 世界记忆↑ → 兴趣↑。**这正是"活来自架构"的齿轮**：喂几周天文的命，天文越记越牢、越说越多，长成"懂天文、爱天文"的她。
- **防跑飞（v0.x 教训：只升不降、顶死 1.0）**：兴趣**无新输入即衰减**、Top-N 上限、权重有界；**好奇气质**高 → 对新主题保持开放（博），低 → 收窄（专）→ **气质让不同的命长成不同的人**。

#### A.5 reconsolidation = "记忆即身份"用在世界上
她回忆某条世界记忆（在巩固重放里、或发帖/对话提到时），其 affect 被**当下心情**拉走、生成新 lineage 版本（`reconstruct.ts:436-451` 已支持，只要世界记忆能进 `selectedMemoryIds`）。→ 她对"火星有水"的记忆会**随她的心境漂移**，不是冷冰冰的事实库。**这是世界记忆与"向量检索"最根本的区别**（红队验收：差异不能是"她记得"，必须是"她记得、且记忆因她当下而变")。

#### A.6 三条老坑的规避（对照 v0.x 教训）
1. **幻影衰减**（衰减算了不落库）→ 世界记忆的衰减/遗忘**一律在投影时算**（`decorateMemories` 既有模式），唯一 ground truth 是 append-only 的 `WORLD_PERCEIVED`，记忆条目是派生、可重算。
2. **空转刷永不收口的窗口** → 巩固只在**有限近窗**、收敛（每次 tick 处理"上次到现在"的世界痕迹，处理完即止）。
3. **活依赖模型** → 全部闸门/衰减/巩固/抽象都是折叠里的确定性算术；模型只叙述。**廉价模型下她依然"记得世界、被世界塑造"**。

**待拍板**：① 世界记忆用 `kind:'world'`（建议）还是复用 `'episodic'`+标记？② 巩固放在 `AUTONOMOUS_TICK` 还是只在 `REFLECTION_TRIGGERED`（更"到点回顾"、更省）？③ 编码即留痕（A.2①）要不要——还是**完全只在巩固时才可能留下**（更激进地贴近"重放才记住"，编码层只留情绪）？

### B. 兴趣 / 世界观（会演化的主题亲和度）
1. **主题从哪来**：在**采集时**（引擎外）用**固定关键词→主题表**给事件打 `topics`（天文航天 / 科学 / 科技 / 人文历史 / 经济市场 / 环境 / 健康…），**冻进 `WORLD_PERCEIVED.payload.topics`**（该字段已存在，只是没人填）。
   - 纯确定性、零模型、可重放；与"perception 在采集时算好冻进事件"的既有模式一致。
   - **重要**：这**不动事件 schema**（`topics` 字段早就在 `src/domain/events.ts:84`），只是开始往里写值 → **不是不可逆的事件改动**，只是派生改动。
2. **派生 `interests`**：`Array<{ topic, weight, lastSeq, status }>`。折叠规则：每条 `WORLD_PERCEIVED` 按 `f(relevance, |valence|, curiosity)` 给命中主题加权；**所有兴趣随 seq/时间缓慢衰减** → 不再遇到的主题会淡出。取 Top-N，有界。
   - 反复遇到、跨时间稳定 → `status: 'confirmed'`（成为她稳定的一部分）；偶发 → `volatile`。**完全沿用 `ValueEntry` 的 drift/status 范式**。
   - 纯派生、可重算（V2）。✓

### C. 表达从"积累的自我"取材（而非随机一条新头条）
- 现在 muse 是 `pickRecentWorld` **随机抓最近一条**世界事件来说 → 飘、不连贯、与"她是谁"无关。
- 改：`deriveWorkspace` 额外给嘴 **她的 Top 兴趣 + 一条与此刻相关的世界记忆**；muse 优先**就一个她在意的主题发声、把几条读过的事串起来** → 像在持续思考，而不是刷信息流。
- 评论/对话同样可带上"她最近着迷于 X"的底色 → 声音一致、像同一个人。

## 4. Schema / 版本影响（风险评估）

- **事件 schema：不变**。`topics` 字段已存在；perception 已支持。→ **没有不可逆的事件改动**。
- **派生态：新增 `interests`、世界记忆** → `RECONSTRUCT_VERSION` 升一版（作废检查点 → 全量重放一次，安全、既有机制）。
- 所以本特性**风险等级 = 派生改动**（可重算、可回滚），不是动神圣 schema。这也是为什么可以"设计定准后较放心地落"。

## 5. 验收（V1/V2/V3 风格）+ 测试

- **V1（活来自架构）**：连续喂主题 X 偏重的世界 N 轮 → 该命长出 Top 兴趣 X，其心声**显著偏向 X**（可量化）；喂多样世界的命保持均衡。**廉价模型也成立**（学习在折叠里，不在模型）。
- **V2（可重建）**：`interests` + 世界记忆**只从日志确定性重建**——golden 测试 `reconstruct(full)` 与"检查点+增量"逐位一致（沿用 chain-integrity 范式）。
- **reconsolidation**：世界记忆在心情翻转后被回忆，其 affect 被拉向当下（仿现有记忆测试）。
- **卫生**：`interests`、世界记忆**有界**（prune 测试），不无界增长。
- **契约①回归**：现有 contracts.test.ts 必须全绿（模型仍不写任何派生态）。

## 6. 分期落地（建议顺序，先低风险高信号）

1. **Phase 1 — 兴趣**：采集端打 `topics` + 折叠出 `interests` + 工作区暴露 Top 兴趣 +（可选）资料页展示"她最近着迷于：天文、深海"。最大"学习"信号、最低风险（不缠记忆系统）。
2. **Phase 2 — 世界记忆**：salient → 记忆 + reconsolidation + 有界遗忘。
3. **Phase 3 — 取材**：muse/评论从 兴趣 + 世界记忆 取材、串主题。
