# 世界学习设计稿（让世界源有长期价值）

> 状态：**设计待拍板**，尚未实现。配套读 `docs/vega-product.md`（北极星）与 `docs/vega-architecture-v1.md`。
> 一句话：现在世界感知是"一次性情绪调味料 + 发帖素材"，**没接进引擎已有的学习机器**。本稿把它接进去——全部走**确定性折叠**（契约①：模型不写状态），让"持续变聪明 / 记忆即身份 / 因你而变"在世界这块也成立。

## 1. 问题（为什么现在浪费）—— 有代码依据

一条 `WORLD_PERCEIVED` 进引擎，`appraiseWorld`（`src/kernel/reconstruct.ts:392`）**只做一件事**：

```
s.valence/arousal/energy += W(=0.06) × val × relevance × sens
```

给情绪轻轻推一下，随时间衰减归零。**它不写任何持久派生结构。** 而引擎早有一整套"沉淀/学习"的机器，世界感知一个都没接：

- **情景记忆 `st.memory`**（带 reconsolidation：回忆被当下情感改写；可压缩成"理解" `SemanticMemory`）——只有**人际消息**会生成记忆（`reconstruct.ts:372`），**世界事件从不生成记忆**。
- **价值 `ValueEntry`（含 status volatile/confirmed + drift）、目标 `Goal`（含 `explore`）、语义记忆**——只由人际/反思喂养。
- **`topics: string[]` 字段从头到尾是空的**（`src/world/sources.ts` 三个 parser 全写 `topics: []`，没人填、没人用）。

结论：她读了一百条新闻，**事件日志里有（ground truth 在），但她的"自我"里一条都没沉淀**。这与第一性原理脱节。

## 2. 必须守住的契约（红线）

1. **契约①**：兴趣 / 世界记忆 / 世界观**全部由确定性符号推理产生**；模型绝不产出它们，只在工作区里**读到**它们、只产对外措辞。
2. **可重算（V2）**：所有新派生结构都能**只从日志确定性重建**；新增的是派生态，不是新的 ground truth。
3. **不引入"模型写的命题信念"**：v1 **不做"观点/立场"**（太接近模型代笔信念）。兴趣 = **定量的主题亲和度**（可确定性计算），不是命题。
4. **有界**：兴趣表、世界记忆都必须有上限/衰减，绝不无界增长（旧版"只升不降"的教训）。

## 3. 方案（三刀，可分期）

### A. 显著世界事件 → 变成"世界记忆"（记忆即身份用在世界上）
在 `appraiseWorld` 里加确定性规则：当 `salience = relevance × |valence|`（好奇高时放宽阈值）超过 θ_world，就 push 一条记忆：

- `content` = 标题（必要时 + 摘要片段）；`affect` = val；`salience` = relevance×|val|。
- `involvedRelationshipIds: []`（无关系人）→ **不进任何"关系的理解"语义压缩**（世界记忆压缩到"兴趣"，见 B）。
- `provenance.status`：salience 高 → `confirmed`，否则 `volatile`；走和人际记忆一样的 lineage。
- **关键收益**：它参与 **reconsolidation**——日后被回忆时按**当下心情**重新染色（"那条火星有水的新闻，现在想起来…"）。这才是真正的"她记得、且记忆会变"。
- **有界**：复用现有遗忘/压缩路径；低 salience 的世界记忆优先被忘/被抽象。

**待拍板**：世界记忆用**新 `kind:'world'`**（干净、好筛）还是**复用 `'episodic'` + 一个 source 标记**（改动小）？

### B. 兴趣 / 世界观（会演化的主题亲和度）
1. **主题从哪来**：在**采集时**（引擎外）用**固定关键词→主题表**给事件打 `topics`（天文航天 / 科学 / 科技 / 人文历史 / 经济市场 / 环境 / 健康…），**冻进 `WORLD_PERCEIVED.payload.topics`**（该字段已存在，只是没人填）。
   - 纯确定性、零模型、可重放；与"perception 在采集时算好冻进事件"的既有模式一致。
   - **重要**：这**不动事件 schema**（`topics` 字段早就在 `src/domain/events.ts:84`），只是开始往里写值 → **不是不可逆的事件改动**，只是派生改动。
2. **派生 `interests`**：`Array<{ topic, weight, lastSeq, status }>`。折叠规则：每条 `WORLD_PERCEIVED` 按 `f(relevance, |valence|, curiosity)` 给命中主题加权；**所有兴趣随 seq/时间缓慢衰减** → 不再遇到的主题会淡出。取 Top-N，有界。
   - 反复遇到、跨时间稳定 → `status: 'confirmed'`（成为她稳定的一部分）；偶发 → `volatile`。**完全沿用 `ValueEntry` 的 drift/status 范式**。
   - 纯派生、可重算（V2）。✓

### C. 表达从"积累的自我"取材（而非随机一条新头条）
- 现在 muse 是 `pickRecentWorld` **随机抓最近一条**世界事件来说 → 飘、不连贯、与"她是谁"无关。
- 改：`deriveWorkspace` 额外给嘴 **她的 Top 兴趣 + 一条与此刻相关的世界记忆**；muse 优先**就一个她在意的主题发声、把几条读过的事串起来** → 像在持续思考，而不是刷信息流。
- 评论/对话同样可带上"她最近着迷于 X"的底色 → 声音一致、像同一个人。

## 4. Schema / 版本影响（风险评估）

- **事件 schema：不变**。`topics` 字段已存在；perception 已支持。→ **没有不可逆的事件改动**。
- **派生态：新增 `interests`、世界记忆** → `RECONSTRUCT_VERSION` 升一版（作废检查点 → 全量重放一次，安全、既有机制）。
- 所以本特性**风险等级 = 派生改动**（可重算、可回滚），不是动神圣 schema。这也是为什么可以"设计定准后较放心地落"。

## 5. 验收（V1/V2/V3 风格）+ 测试

- **V1（活来自架构）**：连续喂主题 X 偏重的世界 N 轮 → 该命长出 Top 兴趣 X，其心声**显著偏向 X**（可量化）；喂多样世界的命保持均衡。**廉价模型也成立**（学习在折叠里，不在模型）。
- **V2（可重建）**：`interests` + 世界记忆**只从日志确定性重建**——golden 测试 `reconstruct(full)` 与"检查点+增量"逐位一致（沿用 chain-integrity 范式）。
- **reconsolidation**：世界记忆在心情翻转后被回忆，其 affect 被拉向当下（仿现有记忆测试）。
- **卫生**：`interests`、世界记忆**有界**（prune 测试），不无界增长。
- **契约①回归**：现有 contracts.test.ts 必须全绿（模型仍不写任何派生态）。

## 6. 分期落地（建议顺序，先低风险高信号）

1. **Phase 1 — 兴趣**：采集端打 `topics` + 折叠出 `interests` + 工作区暴露 Top 兴趣 +（可选）资料页展示"她最近着迷于：天文、深海"。最大"学习"信号、最低风险（不缠记忆系统）。
2. **Phase 2 — 世界记忆**：salient → 记忆 + reconsolidation + 有界遗忘。
3. **Phase 3 — 取材**：muse/评论从 兴趣 + 世界记忆 取材、串主题。
