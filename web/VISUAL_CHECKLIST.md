# 部署前真机检查清单（约 10 分钟）

> 本环境起不了浏览器、更复现不了 iOS WebKit 的渲染怪癖——所以**这一关靠真机**。
> 自动守卫（`npm run lint:ui`）只结构性挡住"边框接缝/颜色漂移"两类回归，像素观感仍需肉眼过。

预览：`cd web && npm run build && npm run preview`（或直接看线上）。手机 + 桌面、浅色 + 深色各过一遍。

## 必查（历史踩坑点）
- [ ] **对话/详情输入栏**：聚焦时外框是**连续完整**的细框；发消息后**不断裂**（iOS 重点，曾反复复发）。
- [ ] **键盘**：弹起时输入栏稳稳钉在键盘正上方，收起即复原（iOS Safari）。
- [ ] **分隔线对齐**：对话/搜索/通知/广场流的行分隔线，左右落在同一条 16px 边。
- [ ] **吸顶页头 / 底部导航**的 1px 边线连续不断（深、浅色都看）。
- [ ] **通知红点**：她主动找你时底栏「通知」亮红点；点进通知页即清。
- [ ] **系统/欢迎消息**是圆形头像（星芒 / 铃），不是小色点。
- [ ] **流动缩放**：不同宽度手机上，标题/留白平滑缩放，无跳变或挤压。
- [ ] **深色模式**：所有页面文字/边线/卡片对比正常，无突兀色块。
- [ ] **二级页一致**：对话「我」气泡中性灰、个人页平铺、三页字号统一。

## 自动守卫（CI 已挡；本地也可跑）
```
cd web && npm run lint:ui   # 接缝守卫（fixed/sticky 禁真 border）+ 令牌守卫（禁硬编码 hex）
cd web && npm run build     # 构建必须绿
```
